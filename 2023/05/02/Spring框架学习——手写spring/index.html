创建简单的bean容器：

概念：

spring包含并管理应用对象的生命周期和配置，spring容器是承载应用对象的载体。

我们可以自行定义bean，每用到应用对象，都会进行“bean的加载，对象创建、对象实例化等等”。bean中也会用到其它的bean。

我们将所有的bean交给spring统一管理，在业务进行的时候由spring帮我们创建对象，管理对象的生命周期。

容器：能存放数据的具体数据结构都叫容器。如：ararayList hashMap等等。

对于spring而言，每个bean都有其独一无二的名称，在应用中会频繁的获取、销毁bean，我们采用哈希表hashmap再合适不过了。

目标：

我们要实现bean的定义、注册、获取。

  • 定义：我们放在BeanDefinition类中。
  • 注册：BeanFactory是产生bean对象的工厂，那么我们注册的bean的信息就在工厂内部保存。所以我们把注册逻辑写入BeanFactory类里。
  • 获取：获取具体的Bean也是从工厂获取。 （不过要依赖于BeanDefinition）。

目标：

我们在上一部分写的bean的注册逻辑非常简单，而且仅仅只是注册了类信息，而不会把类的实例化信息注册到spring。

也就是说，我们从spring获取bean，其实还是bean类自行创建好，放到spring里的map里，并不是spring帮我们创建好的对象。

为了把实例化信息也注册进去，要把bean的class传入到spring容器里。

把 Object bean 改为 Class bean

  • BeanFactory工厂的完善：把beanFactory定义为接口，用AbstractBeanFactory抽象类实现，使用模板模式的设计方式，可以统一归纳通用核心方法的调用逻辑和标准定义，也就很好的控制了后续的实现者不用关心调用逻辑，按照统一方式执行。那么类的继承者只需要关心具体方法的逻辑实现即可。

  • 在继承抽象类 AbstractBeanFactory 后的 AbstractAutowireCapableBeanFactory 就可以实现相应的抽象方法了，因为 AbstractAutowireCapableBeanFactory 本身也是一个抽象类，所以它只会实现属于自己的抽象方法，其他抽象方法由继承 AbstractAutowireCapableBeanFactory 的类实现。这里就体现了类实现过程中的各司其职，你只需要关心属于你的内容，不是你的内容，不要参与。

  • 关于单例 SingletonBeanRegistry 的接口定义实现，而 DefaultSingletonBeanRegistry 对接口实现后，会被抽象类 AbstractBeanFactory 继承。现在 AbstractBeanFactory 就是一个非常完整且强大的抽象类了，也能非常好的体现出它对模板模式的抽象定义。

基于Cglib实现含构造函数的类实例化策略

问题描述：

上一节，我们获取bean对象的方法是createBean，通过反射实例化bean对象。

但是如果bean里有含参构造函数，就会抛出异常：

原因：

使用newInstance方法实例化含有有参构造函数的类对象会报错，是因为newInstance方法会调用类的无参构造函数来创建实例。如果一个类只有有参构造函数而没有无参构造函数，那么newInstance方法无法创建实例，因为它没有提供构造函数参数。

目标：

  1. 从哪合理的把构造函数的入参信息传递到实例化操作？
  2. 怎样实例化含有有参构造函数的对象？

  • 参考 Spring Bean 容器源码的实现方式，在 BeanFactory 中添加 Object getBean(String name, Object... args) 接口，这样就可以在获取 Bean 时把构造函数的入参信息传递进去了。

  • 基于 Java 本身自带的方法 DeclaredConstructor，另外一个是使用 Cglib 来动态创建 Bean 对象。Cglib 是基于字节码框架 ASM 实现，所以你也可以直接通过 ASM 操作指令码来创建对象

    • DeclaredConstructor：是Java反射API中的一个类，表示类的构造函数对象。它继承自Constructor类，因此包含了Constructor类的所有方法，同时也提供了一些额外的方法用于获取构造函数的信息。

      在Java中，每个类都有一个或多个构造函数，用于创建该类的对象。DeclaredConstructor类封装了这些构造函数，允许在运行时通过反射来获取构造函数的详细信息，如参数类型、访问修饰符等。

      常用方法：

      • newInstance(Object... initargs)：使用构造函数来创建一个新的对象，其中initargs是一个可变参数列表，表示构造函数的参数。
      • setAccessible(boolean flag)：设置构造函数的可访问性，如果设置为true，则可以访问私有构造函数，否则不能。
      • getModifiers()：获取构造函数的访问修饰符，返回一个表示修饰符的整数。
      • getParameterTypes()：获取构造函数的参数类型，返回一个表示参数类型的Class对象数组。

    • Cglib：是一个基于ASM开发的高性能字节码增强库，可以在运行时动态地生成指定类的子类，并覆盖其中的方法。因此，Cglib可以实现类似于Java动态代理的功能，同时还可以对类进行更灵活的增强操作。

      Cglib中实例化一个bean对象的过程通常分为以下几步：

      1. 创建Enhancer对象，用于生成子类。
      2. 设置父类和回调方法。
      3. 使用create()方法创建子类对象。

为bean注入属性和依赖对象

假如我们的bean里会有其他的属性，该如何去添加呢？

属性填充是在 Bean 使用 newInstance 或者 Cglib 创建后，开始补全属性信息，那么就可以在类 AbstractAutowireCapableBeanFactory 的 createBean 方法中添加补全属性方法。

目标：

  • 属性填充要在类实例化创建之后，也就是需要在 AbstractAutowireCapableBeanFactory 的 createBean 方法中添加 applyPropertyValues 操作。
  • 属性应该放在bean的beanDefinition里。
  • 填充属性信息还包括了 Bean 的对象类型，需要额外实现BeanReference。
