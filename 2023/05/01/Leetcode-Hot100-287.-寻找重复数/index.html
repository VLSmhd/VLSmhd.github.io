287. 寻找重复数

给定一个包含 n + 1 个整数的数组 nums ，其数字都在 [1, n] 范围内（包括 1 和 n），可知至少存在一个重复的整数。

假设 nums 只有 一个重复的整数 ，返回 这个重复的数 。

你设计的解决方案必须 不修改 数组 nums 且只用常量级 O(1) 的额外空间。

提示：

  • 1 <= n <= 105
  • nums.length == n + 1
  • 1 <= nums[i] <= n
  • nums 中 只有一个整数 出现 两次或多次 ，其余整数均只出现 一次

算法分析：

二分 + 抽屉原理：

参考leetcode题解某位大佬。

抽屉原理简介：

假如有n+1个元素放到n个集合中去，其中必定有一个集合里至少有两个元素。

思路：

对于本题而言，数组nums里有n+1个元素，元素范围[1,...,n]。我们的目标是从元素范围内找出一个数，这个数在nums里至少有一个。

二分查找的一个重要特征：从有序序列找到一个满足条件的元素。

怎样把二分与这题相结合？

“猜数字”：我们利用二分在[1,n]的序列里，寻找中间的数。也就是说，猜[1,n]里最中间那个数是重复数字。这个数的下标为mid

然后遍历nums数组，假如数组里有超过mid数的元素≤该mid元素，那么重复的数字一定会在[1,n]序列里的[left,mid]区间里。（如果遍历一遍输入数组，统计小于 等于 4 的元素的个数，如果小于等于 4 的元素的个数 严格 大于 4 ，说明重复的元素一定出现在整数区间 [1..4]，依然是利用了「抽屉原理」，注意这里加着重号的地方。）

  • 二分指针注意：

    left = mid + 1；

    right = mid；

    保证left与right最后循环结束时重复。
