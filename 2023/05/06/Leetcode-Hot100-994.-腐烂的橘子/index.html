994. 腐烂的橘子

在给定的 m x n 网格 grid 中，每个单元格可以有以下三个值之一：

  • 值 0 代表空单元格；
  • 值 1 代表新鲜橘子；
  • 值 2 代表腐烂的橘子。

每分钟，腐烂的橘子 周围 4 个方向上相邻 的新鲜橘子都会腐烂。

返回 直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。如果不可能，返回 -1 。

算法分析：

广度优先搜索BFS：

  • 辅助队列：针对本题，队列里存放烂橘子的相关信息即可，每个队列元素是数组型，存放烂橘子下标。每遍历一个烂橘子就把它移出队列，然后将它感染的橘子加入进来，然后把烂橘子遍历完，感染完。

  • 初始化：先遍历一下网格，把好橘子的个数记录下来，记为count，然后将初始的烂橘子信息（横坐标、纵坐标）添加到队列。

  • 广搜逻辑：

    • 外循环

      • 判断条件：count > 0(还有好橘子) && !queue.isEmpty() (烂橘子还在感染ing)
      • 外循环的每一轮，对应ans++

    • 内循环：

      因为每轮感染，感染者都是上一轮中队列剩下的烂橘子（第一轮循环可以看成初始化轮剩下的烂橘子）。这时候队列有个size，即为内循环的终止下标。内循环就负责处理广搜逻辑，解决上一轮剩下的烂橘子。

      • 搜索方向：针对网格型进行广搜，只要想象你站在某个格子上，有四条腿踏向你的四周：上、下、左、右。

      • 每个方向上的处理逻辑：

        如果对应方向上的橘子a存在且还未被感染，则感染他。感染逻辑：将a置为坏橘子，count--，添加到队列中为了下一轮感染。
