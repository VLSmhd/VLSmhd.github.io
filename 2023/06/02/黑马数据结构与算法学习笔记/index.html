局部性原理：

空间局部性：

  • cpu 读取内存（速度慢）数据后，会将其放入高速缓存（速度快）当中，如果后来的计算再用到此数据，在缓存中能读到的话，就不必读内存了
  • 缓存的最小存储单位是缓存行（cache line），一般是 64 bytes，因此最少读 64 bytes 填满一个缓存行，因此读入某个数据时也会读取其临近的数据（物理上的临近），这就是所谓空间局部性

实现代码：

  • 搜索区间：根据初始化前置条件，得知搜索区间就是[0,a.length-1]，注意是闭区间。

  • 循环条件i <= j：满足条件的时候，说明区间[i,j]内还有未比较的元素，i和j所指的元素也可能是结果。

    • 那如果循环条件改成i < j成立不？

      修改条件后，循环终止的状态就是i = j的状态，此时基础版代码还会求一次mid也就是在i和j同时指的位置处，然后进行最后一次比较，看mid是否是target。但是修改条件了，最后一次比较没了。

      案例：

      该案例寻找元素15，很明显它是存在于数组中的，但是在二分的过程中，将target与15进行比较是放在最后一次比较进行的，此时恰好循环条件改为（i < j），跳过了最后一次比较，所以返回结果不是15的下标而是-1（不存在该元素）。

  • 中间索引计算的问题：

    当i和j都很大的时候（接近Integer.MAX_VALUE），(i+j)会溢出，此时java底层的(i+j)会变成负数，二分查找就不能进行了。

    • 原因：

      看下面两个二进制数：

      (i+j)溢出，实际等于3221225470，但是其二进制位达到了32位，于是java会把第一位看成符号位，这样大数3221225470就变成负的了。

    • 改进：

      将(i+j)/2 改为(i+j)>>>1

      (i+j)/2并不会动符号位，所以是负数/2不符合。

      (i+j)>>>1符号位也会跟着移动，然后前面补0，这样就相当于数本身没变化，右移一位等价于除以2，结果正确。

递归

概述

定义

计算机科学中，递归是一种解决计算问题的方法，这类问题通常有共性，就是需要重复的对子问题进行相同的处理。

特点

  1. 自己调用自己，如果说每个函数对应着一种解决方案，自己调用自己意味着解决方案是一样的（有规律的）
  2. 每次调用，函数处理的数据会较上次缩减（子集），而且最后会缩减至无需继续递归
  3. 内层函数调用（子集处理）完成，外层函数才能算调用完成
  4. 深入到最里层叫做递
  5. 从最里层出来叫做归
  6. 在递的过程中，外层函数内的局部变量（以及方法参数）并未消失，归的时候还可以用到

递归要素
